Opioid Epidemic Analysis¶

By Kyle Hassold and Devesh Agarwal¶

In this tutorial we took a dataset which summarized prescription data for 250 different drugs both opioids and non-opioid and data for overdose deaths in each state. The dataset was obtained from Kaggle.com and contains 25,000 unique prescriptions in the year 2014. The Kaggle dataset is a small subset of the data available from cms.gov which has details of about 24 million prescriptions which are written in 2014. In our tutorial we wanted to analyze the connection between prescriber's, opioid prescriptions, and opioid overdoses.

Motivation:¶

We wanted to learn more about the Opioid epidemic which has occurred over the last 20 years and see how opiate prescriptions affect the overdoses. Since the opioid crisis is so vast in the U.S. with 136 people dying each day from opioid overdose according to the CDC, we wanted to learn how often opioids were being prescribed to patients. We choose our data in order to find out which states were prescribing opioids the most, which states had the most deaths from opioid overdoses and which specialties were most frequently prescribing opioids to their patients.

Datset 2: Overdoses¶

overdose.csv: Overdose data (odData) which gives the state, the population per state, and the number of opioid related deaths in that state.

The imported data contained commas in the deaths and the population columns which made them read as strings when imported. To turn the deaths and populations to be numerical values rather than string phrases, we removed the commas leaving just the numbers which pandas converted to a number using the to_numeric function.

For later use, we added a column which found the number of opioid deaths per 100,000 people in the state by dividing the number of overdose deaths by the population and multiplying by 100,000.

In [ ]:
odData = pd.read_csv("/content/overdoses.csv")

# Remove commas and change data type to numbers rather than strings for Population and Deaths columns
odData["Deaths"] = odData["Deaths"].replace(',','', regex=True).apply(pd.to_numeric,errors='coerce')
odData["Population"] = odData["Population"].replace(',','', regex=True).apply(pd.to_numeric,errors='coerce')

# Create new column for Opioid Overdoses per 100,000 people in the state
odData["deathPer"] = odData["Deaths"]/odData["Population"]*100000

Dataset 3: Prescriber Info¶

Prescriber Data (presData) which gives data about the Gender of the Prescriber, what state they practice in, the credentials and specialty. In addition, it provides the number of prescriptions given of 250 common drugs prescribed and a boolean marker if the individual prescribed over 10 opioids in a year.

We added multiple columns to this dataset for easier use in the future. We began by creating a column for the number of prescriptions a given individual had in the year, then created another column for the number opioid prescriptions an individual had in a year and finally created a column which calculated the proportion of opioid prescriptions to regular prescriptions the individual did that year.

Linear Regression¶

To test the effect of various pieces of data about prescribers on their proportion of opioid prescriptions, we used a linear regression. As inputs to the regression model, we used gender, whether they are an opioid prescriber as defined above, dummy columns for specialties, and dummy columns for states.

When conducting this linear regression, it was found that including dummy column for states had very little effect on the regression score and each state has a very small coefficient (meaning the state value did not change the output much). Increasing the number of dummy columns for specialties, does drastically increase the regression score to the 0.46 that it is currently at. Ultimately, the regression score of 0.46 is still small meaning there is not much correlation between the values of gender, opioid prescriber, specialties, and state to the opioid prescription proportion.
